Working

  • The Control Unit works by following the instruction cycle, which includes three main steps: Fetch, Decode, and Execute. These steps are repeated continuously for every instruction in a program.

  • In the Fetch phase, the Control Unit retrieves the next instruction from memory using the Program Counter (PC). The instruction is then stored in the Instruction Register (IR).

  • In the Decode phase, the Control Unit interprets the instruction to determine the required operation. It identifies what action needs to be performed and which components are involved.

  • In the Execute phase, the Control Unit generates appropriate control signals to carry out the operation. It directs the ALU, registers, and memory to complete the instruction.

  • After execution, the Program Counter is updated to point to the next instruction, and the cycle repeats.
